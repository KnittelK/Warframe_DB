# Warframe Mod Browser — Project Guide

## What this is
A fully static Warframe mod browser hosted on GitHub Pages at `https://knittelk.github.io/Warframe_DB/`. No backend. All data is pre-merged JSON served alongside the HTML/CSS/JS.

## Directory layout
```
docs/               Static site root — what gets deployed to GitHub Pages
  index.html
  css/style.css
  js/app.js
  data/
    mods.json       Generated by CI — DO NOT hand-edit (overwritten nightly)
    meta.json       Build metadata (lastUpdated, modCount, deAvailable)
scripts/
  merge-mods.js    Node 18+ script that fetches and merges mod data
.github/workflows/
  update-mods.yml  Nightly data fetch (04:00 UTC) + manual trigger
  deploy-pages.yml Deploys docs/ to GitHub Pages
```


## Data pipeline
1. `update-mods.yml` runs nightly: fetches DE Public Export + warframestat.us, merges, writes `docs/data/mods.json` + `docs/data/meta.json`, commits, pushes.
2. The push is supposed to trigger `deploy-pages.yml` — **but this only works with a PAT, not the default GITHUB_TOKEN** (see Known Issues below).

Data sources:
- `https://content.warframe.com/PublicExport/Manifest/ExportUpgrades.json` — canonical stats (DE official, sometimes unavailable)
- `https://api.warframestat.us/mods` — drop tables, images, descriptions

## Known Issues

### Deploy not re-triggered after data update
`update-mods.yml` pushes with `GITHUB_TOKEN`. GitHub Actions will not fire another workflow's `on: push` trigger from a `GITHUB_TOKEN` push. This means after each nightly data update, the deployed site is NOT automatically redeployed with the new data.

**Fix options:**
- Add a `workflow_dispatch` call or `workflow_run` trigger in `deploy-pages.yml` to run after `update-mods.yml` completes.
- Use a repository PAT secret instead of `GITHUB_TOKEN` for the push in `update-mods.yml`.
- Merge both workflows: fetch data and deploy in a single workflow.

## Local development
```bash
# Serve the site locally
cd docs && python3 -m http.server 8000

# Run the data merge manually (requires Node 18+)
node scripts/merge-mods.js
```

## Deployment
Deployed via GitHub Actions (`deploy-pages.yml`) using `actions/deploy-pages`. GitHub Pages source must be set to **GitHub Actions** (not a branch) in repo Settings → Pages.
